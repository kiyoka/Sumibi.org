

             プロトコル設計メモ

            Protocol design memo
          


1. プロトコルの種類
  - 標準入出力  ... uim,sumibi_soap.cgi,Emacsとの通信で使用する。
  - REST        ... Emacs + curlとの通信で使用する。
  - SOAP        ... uim等のフレームワークからの通信で使用する。


2. 標準入出力のプロトコル
  - 基本ルール
    入力:
      [関数名]  [パラメータ]
      *  関数名とパラメータの間は1文字以上のスペースとする。

    出力:
      1行目       ( 正常に終了したかどうかを返す )
        ok/error
      2行目以降   ( 関数の結果を返す。複数行の場合も有りえる )

      *  プログラムの終了が出力の終わりとする。


3. RESTのプロトコル

  - http POSTメソッドの場合、RESTとして動作する。
    入力引数
      string ... 変換ソース文字列
      encode ... 返却結果の文字エンコード

    変換結果のS式をそのまま文字列で返す。

  - http GET メソッドの場合、Sumibi Serverの状況を
    人間が読める形にしたHTMLページを返す。
